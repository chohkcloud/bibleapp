# 📝 문서 명명 규칙 (Document Naming Convention)

> Claude Code가 문서 생성 시 반드시 따라야 하는 명명 규칙

---

## 1. 문서명 형식

```
[카테고리]_[파일명]_[연월일]_[시분].md
```

### 형식 설명

| 순서 | 항목 | 형식 | 예시 |
|------|------|------|------|
| 1 | 카테고리 | 한글 대문자 | `신규생성`, `디버그`, `메뉴얼` |
| 2 | 파일명 | 한글 또는 영문 (공백 없음) | `DB설계`, `화면설계`, `API명세` |
| 3 | 연월일 | YYYYMMDD | `20260101` |
| 4 | 시분 | HHMM (24시간, KST) | `1430` |

### 구분자
- 각 항목은 언더스코어(`_`)로 구분
- 파일명 내부 단어는 붙여쓰기 또는 카멜케이스

---

## 2. 카테고리 정의

| 카테고리 | 코드 | 용도 | 사용 시점 |
|----------|------|------|----------|
| **신규생성** | `신규생성` | 새로운 기능, 설계, 구조 문서 | 최초 문서 작성 시 |
| **디버그** | `디버그` | 버그 수정, 오류 해결 기록 | 문제 해결 과정 기록 시 |
| **메뉴얼** | `메뉴얼` | 사용법, 가이드, 튜토리얼 | 사용 방법 안내 시 |
| **수정** | `수정` | 기존 문서/코드 변경 사항 | 기존 내용 업데이트 시 |
| **분석** | `분석` | 요구사항, 기술 분석 | 분석 결과 정리 시 |
| **회의록** | `회의록` | 미팅, 논의 내용 기록 | 회의 내용 정리 시 |

---

## 3. 파일명 예시

### 설계 관련
| 파일명 | 설명 |
|--------|------|
| `DB설계` | 데이터베이스 스키마 설계 |
| `화면설계` | UI/UX 화면 설계 |
| `API설계` | API 엔드포인트 설계 |
| `아키텍처설계` | 시스템 구조 설계 |
| `네비게이션설계` | 화면 흐름 설계 |

### 구현 관련
| 파일명 | 설명 |
|--------|------|
| `테이블생성` | DB 테이블 생성 SQL |
| `컴포넌트구현` | React 컴포넌트 구현 |
| `서비스구현` | 비즈니스 로직 구현 |
| `훅구현` | Custom Hook 구현 |
| `스타일구현` | CSS/스타일 구현 |

### 기타
| 파일명 | 설명 |
|--------|------|
| `에러해결` | 버그/에러 해결 기록 |
| `성능개선` | 성능 최적화 내용 |
| `리팩토링` | 코드 리팩토링 내용 |
| `배포가이드` | 배포 절차 안내 |
| `설치가이드` | 설치 방법 안내 |

## 5. 시간 기준

### 한국 표준시 (KST, UTC+9)

```javascript
// JavaScript로 현재 한국 시간 얻기
const now = new Date();
const kstTime = new Date(now.getTime() + (9 * 60 * 60 * 1000));

// 형식: YYYYMMDD_HHMM
const dateStr = kstTime.toISOString().slice(0, 10).replace(/-/g, '');
const timeStr = kstTime.toISOString().slice(11, 16).replace(':', '');
const filename = `신규생성_파일명_${dateStr}_${timeStr}.md`;
```

### 시간대 변환 참고
| UTC | KST (한국) |
|-----|-----------|
| 00:00 | 09:00 |
| 06:00 | 15:00 |
| 12:00 | 21:00 |
| 15:00 | 24:00 (다음날 00:00) |

## 6. docs_index.md 등록 규칙

### ⚠️ 필수 사항
> **모든 생성된 문서는 반드시 `docs_index.md`에 등록해야 한다.**

### 등록 형식

```markdown
| # | 문서명 | 카테고리 | 용도 | 생성일시 | 위치 |
|---|--------|----------|------|----------|------|
| N | 파일명.md | 카테고리 | 문서 설명 | YYYYMMDD_HHMM | 전체 경로 |
```

### 등록 예시

```markdown
| # | 문서명 | 카테고리 | 용도 | 생성일시 | 위치 |
|---|--------|----------|------|----------|------|
| 7 | 신규생성_메모기능_20260101_1430.md | 신규생성 | 메모 기능 상세 설계 | 20260101_1430 | `C:/claude/claudeproject/bibleapp/docs/신규생성_메모기능_20260101_1430.md` |
| 8 | 디버그_DB연결오류_20260102_0930.md | 디버그 | SQLite 연결 오류 해결 | 20260102_0930 | `C:/claude/claudeproject/bibleapp/docs/디버그_DB연결오류_20260102_0930.md` |
```

## 7. Claude Code 작업 절차

### 문서 생성 시 필수 단계

```
┌─────────────────────────────────────────────────────────────┐
│                    문서 생성 워크플로우                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Step 1: 카테고리 결정                                      │
│  ├── 새 기능/설계 → 신규생성                                │
│  ├── 버그 수정 → 디버그                                     │
│  ├── 사용 가이드 → 메뉴얼                                   │
│  ├── 기존 문서 변경 → 수정                                  │
│  └── 분석 내용 → 분석                                       │
│                                                             │
│  Step 2: 파일명 결정                                        │
│  └── 내용을 간결하게 표현 (한글 또는 영문)                  │
│                                                             │
│  Step 3: 현재 한국 시간 확인                                │
│  └── 연월일: YYYYMMDD, 시분: HHMM (KST)                    │
│                                                             │
│  Step 4: 문서 파일 생성                                     │
│  └── 경로: C:/claude/claudeproject/bibleapp/docs/           │
│                                                             │
│  Step 5: docs_index.md에 등록 ⚠️ 필수                       │
│  └── 문서 목록 테이블에 새 행 추가                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### Claude Code 실행 예시

```
[사용자 요청]
"메모 암호화 기능 설계 문서를 만들어줘"

[Claude Code 실행]
1. 카테고리: 신규생성 (새 기능 설계)
2. 파일명: 메모암호화설계
3. 현재 시간: 2026-01-01 14:30 KST
4. 최종 파일명: 신규생성_메모암호화설계_20260101_1430.md
5. 저장 경로: C:/claude/claudeproject/bibleapp/docs/신규생성_메모암호화설계_20260101_1430.md
6. docs_index.md 업데이트: 새 문서 정보 추가
```

## 8. 문서 내부 헤더 템플릿

모든 문서는 아래 헤더로 시작해야 한다:

```markdown
# [문서 제목]

> [문서 한 줄 설명]

---

## 문서 정보

| 항목 | 내용 |
|------|------|
| 문서명 | [전체 파일명] |
| 카테고리 | [신규생성/디버그/메뉴얼/수정/분석] |
| 생성일시 | [YYYY-MM-DD HH:MM KST] |
| 작성자 | Claude |
| 버전 | v1.0.0 |
| 위치 | [전체 경로] |

---

## 목차

1. [섹션1]
2. [섹션2]
3. [섹션3]

---

[본문 내용]
```
